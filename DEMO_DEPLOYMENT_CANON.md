# BackOfHouseOS Demo Deployment Canon

## Core Rule

Real systems create truth. Public demos show the shape.

## Deployment Rule

Public demos live in GitHub and auto-deploy through Vercel after the project is connected.

## Monorepo Rule

Use one repo with many demo folders.

Each Vercel project points to a specific root directory.

## Safety Rule

No private runtime data enters public demo folders.

Public demos use sample data only. Real systems may run locally/private.

## Notion Rule

Only link confirmed public URLs.

Never link localhost, ngrok, raw folders, raw DBs, or unreviewed previews.

## Demo Status Labels

- public-safe sample demo
- sanitized static demo
- case-study mock
- future concept
- private/local only
